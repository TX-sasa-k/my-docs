GitHub Actions のワークフロー構文（公式見出し準拠版）

対象: https://docs.github.com/ja/actions/reference/workflows-and-actions/workflow-syntax
この文書は、公式ページの章立てに合わせて参照しやすく整理した保存版です。

注意: 公式ドキュメントは更新されるため、最終的な正本は必ず公式サイトを確認してください。

name

ワークフローの表示名を指定します。省略可能です。

name: CI

run-name

個々のワークフロー実行に表示する名前を指定します。式を使った動的な命名が可能です。

run-name: Deploy by @${{ github.actor }}

on

ワークフローを実行するイベントを定義します。

on: [push, pull_request]

on.<event_name>.types

イベントのアクション種別（activity type）で絞り込みます。

on:
  pull_request:
    types: [opened, synchronize, reopened]

on.<pull_request|pull_request_target>.<branches|branches-ignore>

対象ブランチ名でフィルタリングします。

on:
  pull_request:
    branches: [main, 'release/*']

on.<push|pull_request|pull_request_target>.<paths|paths-ignore>

変更ファイルパスで実行対象を制御します。

on:
  push:
    paths:
      - 'src/**'
      - '!src/experimental/**'

on.workflow_call

再利用可能ワークフローとして呼び出されるためのトリガーです。

on.workflow_call.inputs

呼び出し元から受け取る入力を定義します。

on:
  workflow_call:
    inputs:
      environment:
        required: true
        type: string

on.workflow_call.inputs.<input_id>.type

入力値の型（例: string, number, boolean）を指定します。

on.workflow_call.outputs

再利用可能ワークフローが呼び出し元へ返す出力を定義します。

on.workflow_call.secrets

呼び出し元から受け取るシークレットを定義します。

on.workflow_dispatch

手動実行を有効化します（Actions タブ等から実行）。

on.workflow_dispatch.inputs

手動実行時の入力フォーム項目を定義します。

on.workflow_dispatch.inputs.<input_id>.type

入力の UI/型（choice, boolean, environment など）を指定します。

permissions

GITHUB_TOKEN の権限を設定します。ワークフロー全体またはジョブ単位で指定可能です。

permissions:
  contents: read
  pull-requests: write

env

環境変数をワークフロー全体で定義します（ジョブ/ステップで上書き可）。

env:
  NODE_ENV: test

defaults

run ステップのデフォルト設定を定義します。

defaults.run

defaults:
  run:
    shell: bash
    working-directory: ./app

defaults.run.shell

既定のシェルを指定します。

defaults.run.working-directory

既定の作業ディレクトリを指定します。

concurrency

同一グループの実行を制御し、重複実行を抑止できます。

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs

ワークフローを構成するジョブ群です。

jobs.<job_id>

ジョブ定義の識別子。英数字と _ を使って定義します。

jobs.<job_id>.name

ジョブの表示名を指定します。

jobs.<job_id>.permissions

ジョブ単位で GITHUB_TOKEN 権限を上書きします。

jobs.<job_id>.needs

依存ジョブを指定し、完了後に実行します。

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - run: npm test
  deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - run: echo deploy

jobs.<job_id>.if

ジョブの実行条件を指定します。

jobs.<job_id>.runs-on

実行ランナーを指定します（GitHub-hosted または self-hosted）。

jobs.<job_id>.environment

デプロイ先環境（保護ルールや環境シークレット）を指定します。

jobs.<job_id>.concurrency

ジョブ単位の同時実行制御です。

jobs.<job_id>.outputs

ジョブの出力を定義し、後続ジョブで参照できます。

jobs.<job_id>.env

ジョブ内で利用する環境変数を定義します。

jobs.<job_id>.defaults

ジョブ内の run デフォルト設定です。

jobs.<job_id>.steps

ジョブを構成するステップ群です。

jobs.<job_id>.timeout-minutes

ジョブのタイムアウト時間（分）を指定します。

jobs.<job_id>.strategy

マトリックス実行などの戦略設定です。

jobs.<job_id>.strategy.matrix

複数条件の組み合わせでジョブを展開します。

jobs.<job_id>.strategy.matrix.include

追加の組み合わせを定義します。

jobs.<job_id>.strategy.matrix.exclude

不要な組み合わせを除外します。

jobs.<job_id>.continue-on-error

ジョブ失敗時にワークフロー全体を失敗扱いにしない設定です。

jobs.<job_id>.container

ジョブ全体をコンテナ内で実行します。

jobs.<job_id>.container.image

使用するコンテナイメージを指定します。

jobs.<job_id>.container.credentials

コンテナレジストリ認証情報を指定します。

jobs.<job_id>.container.env

ジョブコンテナ用環境変数です。

jobs.<job_id>.container.ports

公開ポートを指定します。

jobs.<job_id>.container.volumes

マウントするボリュームを指定します。

jobs.<job_id>.container.options

追加の Docker 実行オプションを指定します。

jobs.<job_id>.services

ジョブで利用するサービスコンテナ（DB など）を定義します。

jobs.<job_id>.services.<service_id>.image

サービスコンテナのイメージを指定します。

jobs.<job_id>.services.<service_id>.credentials

サービスコンテナのレジストリ認証情報です。

jobs.<job_id>.services.<service_id>.env

サービスコンテナ環境変数です。

jobs.<job_id>.services.<service_id>.ports

サービスコンテナの公開ポートです。

jobs.<job_id>.services.<service_id>.volumes

サービスコンテナのボリューム設定です。

jobs.<job_id>.services.<service_id>.options

サービスコンテナの追加オプションです。

jobs.<job_id>.uses

再利用可能ワークフローを呼び出します。

jobs.<job_id>.with

呼び出し先ワークフローへ入力を渡します。

jobs.<job_id>.with.<input_id>

個別入力値を指定します。

jobs.<job_id>.secrets

呼び出し先ワークフローへシークレットを渡します。

jobs.<job_id>.secrets.<secret_id>

個別シークレット値を指定します。

jobs.<job_id>.secrets.inherit

利用可能なシークレットを一括継承します。

steps（jobs.<job_id>.steps）

jobs.<job_id>.steps[*].id

ステップ識別子。後続ステップから出力参照に使います。

jobs.<job_id>.steps[*].if

ステップ実行条件です。

jobs.<job_id>.steps[*].name

ステップの表示名です。

jobs.<job_id>.steps[*].uses

アクションまたは再利用可能アクションを呼び出します。

jobs.<job_id>.steps[*].run

シェルコマンドを実行します。

jobs.<job_id>.steps[*].working-directory

ステップ実行時の作業ディレクトリです。

jobs.<job_id>.steps[*].shell

ステップで使用するシェルです。

jobs.<job_id>.steps[*].with

uses ステップに渡す入力です。

jobs.<job_id>.steps[*].with.args

アクション引数（Docker アクション等）を指定します。

jobs.<job_id>.steps[*].with.entrypoint

Docker アクションのエントリポイントを上書きします。

jobs.<job_id>.steps[*].env

ステップ単位の環境変数です。

jobs.<job_id>.steps[*].continue-on-error

ステップ失敗時にジョブ継続を許可します。

jobs.<job_id>.steps[*].timeout-minutes

ステップのタイムアウト（分）です。

GitHub Actions の式（Expressions）

${{ ... }} 形式で評価します。 代表コンテキスト: github, env, vars, secrets, matrix, needs, steps。 

if: ${{ github.event_name == 'pull_request' && startsWith(github.head_ref, 'feature/') }}

利用上の注意（要点）

  • 権限は最小化（permissions）する
  • シークレットをログへ出力しない
  • 外部入力をシェルへ渡すときはエスケープする
  • 重複実行は concurrency で抑制する
